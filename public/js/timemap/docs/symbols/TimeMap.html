Class TimeMap

The TimeMap object holds references to timeline, map, and datasets. This will create the visible map, but not the timeline, which must be initialized separately.
Defined in: timemap.js.

<static> {TimeMap} TimeMap.init(config)
Intializes a TimeMap.

This is an attempt to create a general initialization script that will work in most cases. If you need a more complex initialization, write your own script instead of using this one.

The idea here is to throw all of the standard intialization settings into a large object and then pass it to the TimeMap.init() function. The full data format is outlined below, but if you leave elements off the script will use default settings instead.

Call TimeMap.init() inside of an onLoad() function (or a jQuery $.(document).ready() function, or whatever you prefer). See the examples for usage.

Parameters:
{Object} config
Full set of configuration options. See examples/timemapinit_usage.js for format.
Returns:
{TimeMap} The initialized TimeMap object, for future reference
